# Unity-Angular Bridge — Example Project

This example demonstrates how to use the **Unity-Angular-Bridge** code-generation
tools in an Angular 21 application with **OnPush** change detection and
**Signals / Computed**.

It includes:
- **`angular-unity-example/`** — Angular 21 app with a Unity WebGL viewport
- **`unity-project/`** — Unity project with a `SceneManager` MonoBehaviour using the bridge

## Quick Start

```bash
cd example/angular-unity-example
npm install
npm start
```

Open **http://localhost:4200** in your browser.

> The app automatically detects whether a real Unity WebGL build is available.
> If not, a mock Unity instance is used so the demo still works.

---

## Adding a Real Unity WebGL Build

1. **Open Unity Hub** and create a new project (Unity 2022.3 LTS or later, 3D template).

2. **Copy the bridge scripts** from the repository root into your Unity project:
   ```
   AngularExposedAttribute.cs   → Assets/Scripts/UnityAngularBridge/
   AngularExposedExport.cs      → Assets/Scripts/UnityAngularBridge/
   JSLibExport.cs               → Assets/Scripts/UnityAngularBridge/
   JSLibVariable.cs             → Assets/Scripts/UnityAngularBridge/Models/
   StringArrayAttribute.cs      → Assets/Scripts/UnityAngularBridge/
   ```
   (These are already set up in `unity-project/Assets/Scripts/UnityAngularBridge/`)

3. **Copy the example scene script:**
   ```
   unity-project/Assets/Scripts/SceneManager.cs → Assets/Scripts/
   ```

4. **Create a scene** with a GameObject named **"SceneManager"**, attach the script,
   and add a few 3D objects.

5. **Build for WebGL:**
   - File → Build Settings → WebGL → Switch Platform
   - Player Settings → Publishing Settings → Compression Format → **Disabled**
   - Set the build name to **`unity-build`**
   - Click **Build**

6. **Copy the build output** into the Angular app's public folder:
   ```bash
   cp -r <unity-build-output>/* angular-unity-example/public/unity-build/
   ```

7. **Start the Angular app** — the viewport will automatically load the real Unity build.

### Generated Files (from Unity)

These files are **auto-generated** by the Unity C# scripts each time you compile
your Unity project. In this example they contain sample methods for a
"SceneManager" GameObject.

| File | Generator | Purpose |
|------|-----------|---------|
| `unity-client.ts` | `AngularExposedExport.cs` | Typed methods to **call Unity** from Angular via `SendMessage()` |
| `unity-jslib-exported.service.ts` | `JSLibExport.cs` | RxJS observables that **emit** when Unity sends events to Angular |

## Communication Patterns

### Angular → Unity (via `UnityClient`)

C# methods marked with `[AngularExposed]` are exported to `UnityClient.ts`:

```csharp
// Unity C# — SceneManager.cs
[AngularExposed(gameObjectName: "SceneManager")]
public void LoadObject(string objectId) { /* ... */ }
```

Generated TypeScript:

```typescript
// unity-client.ts (auto-generated)
public sceneManager_LoadObject(unityInstance: IUnityInstance, objectId: string): void {
  unityInstance?.SendMessage("SceneManager", "LoadObject", objectId);
}
```

Usage with **Signals** in the bridge service:

```typescript
// unity-bridge.service.ts
readonly isConnected = computed(() => !!this.unityInstance());

loadObject(objectId: string): void {
  const instance = this._unityInstance();
  if (instance) {
    this.unityClient.sceneManager_LoadObject(instance, objectId);
  }
}
```

### Unity → Angular (via `UnityJSLibExportedService`)

C# methods marked with `[DllImport("__Internal")]` (and optionally
`[StringArrayAttribute]`) are exported as observables:

```csharp
// Unity C# — SceneManager.cs
[DllImport("__Internal")]
[StringArrayAttribute]
private static extern void SendObjectsList(string objectIds);

public void ResetScene() {
  SendObjectsList(string.Join("|", objectIds));
}
```

Generated TypeScript:

```typescript
// unity-jslib-exported.service.ts (auto-generated)
public sendObjectsList$: Observable<string[]> = sendObjectsListSubject.asObservable();
```

Usage with **Signals** in the bridge service:

```typescript
// unity-bridge.service.ts
readonly objectsList = toSignal(this.jsLibService.sendObjectsList$, { initialValue: [] });
readonly objectCount = computed(() => this.objectsList().length);
```
